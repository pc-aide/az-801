Question 12

You have a failover cluster named Cluster1 that hosts an application named App1.

The General tab in App1 Properties is shown in the General exhibit.

The Failover tab in App1 Properties is shown in the Failover exhibit.

You need to ensure that when you start Server1, App1 continues to run on Server2.

Solution: From the General settings, you move Server2 up. Does this meet the goal?

Explanation:

To actually change the priority, you need to tick the box. Simply moving the server up without ticking the box does not change the priority, and the order will revert. If you move the server up and tick the box, then the change is effective. The Prevent failback setting would ensure that the application does not fail back to Server1 if Server1 becomes available again.
