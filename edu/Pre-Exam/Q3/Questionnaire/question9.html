Question 9

You have a Hyper-V failover cluster named Cluster1 at a main datacenter. Cluster1 contains two nodes that have the Hyper-V server role installed. Cluster1 hosts 10 highly available virtual machines. You have a cluster named Cluster2 in a disaster recovery site. Cluster2 contains two nodes that have the Hyper-V server role installed.

You plan to use Hyper-V Replica to replicate the virtual machines from Cluster1 to Cluster2.

What should you do? To answer, select the appropriate options in the answer area.

Cluster role to create on Cluster2:

Replication target name to specify:

Correct Answer:

Cluster role to create on Cluster2: Hyper-V Replica Broker
Replication target name to specify: The name of the Hyper-V Replica Broker

Explanation:

For the virtual machines to failover correctly between clusters using Hyper-V Replica, you must create a Hyper-V Replica Broker role in the cluster. This role manages the replication of virtual machines and ensures they are appropriately tracked and managed. The target name should also be the Hyper-V Replica Broker's name, as it coordinates the failover process.
