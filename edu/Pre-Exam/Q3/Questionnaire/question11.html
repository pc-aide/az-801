Question 11

You have a Hyper-V failover cluster named Cluster1 that uses a cloud witness. Cluster1 hosts a virtual machine named VM1 that runs Windows Server.

You need to fail over VM1 automatically to a different node when a service named Service1 on VM1 fails.

What should you do on Cluster1 and VM1? To answer, select the appropriate options in the answer area.

Cluster1:

VM1:

Correct Answers:

Cluster1: Configure monitoring of the VM1 cluster role

VM1: Configure the Recovery settings of Service1

Explanation:

To automatically fail over VM1 to a different node when Service1 fails, you need to configure the cluster to monitor the VM1 role and also configure the service's recovery settings. This ensures that the cluster will take appropriate action based on the service failure on VM1.
