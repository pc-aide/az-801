Question 3

You have a failover cluster named Cluster1 that hosts an application named App1.

The General tab in App1 Properties is shown in the General exhibit.

The Failover tab in App1 Properties is shown in the Failover exhibit.

Server1 shuts down unexpectedly.

You need to ensure that when you start Server1, App1 continues to run on Server2.

Solution: You pause the Server1 node in Cluster1 and then start Server1. Does this meet the goal?



Explanation: If Server1 is paused and Server2 fails or requires a restart, Server1 will not be able to assume the roles until it is resumed. You cannot pause before shutdown because the server shut down unexpectedly. Always pause and drain clustered servers before taking them offline for maintenance.

Correct Answer: B
